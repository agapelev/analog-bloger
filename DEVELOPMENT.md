# 📖 Летопись Разработки Цитадели (Shekinah Blog)

Этот документ фиксирует процесс создания проекта с нуля для будущих поколений и восстановления после "великих откатов".

## 🏗️ Архитектура Проекта

### 1. Маршрутизация (File-based Routing)
Мы используем структуру Analog.js в папке `src/app/pages`:
- `index.page.ts`: Корневой редирект на `/blog`.
- `blog/index.page.ts`: Главная лента постов.
- `blog/[slug].page.ts`: Динамическая страница поста из Markdown.
- `tags/[tag].page.ts`: Фильтрация контента по тегам.

### 2. Компоненты (Holy Components)
Все кастомные элементы хранятся в `src/app/components/`:
- **LogoComponent**: Священный логотип "Shekinah Cloud Mission". Использует Standalone-подход и мистические SVG-анимации.

### 3. Стилизация (Visual Identity)
- **Цветовая схема:** Основной фон `#0f011a` (Deep Purple), акценты `cyan-400` и `fuchsia-500`.
- **Шрифты:** Serif для заголовков (Lora/Georgia), Mono для метаданных.
- **Градиенты:** Используются CSS-анимации `animate-gradient-text-mystic`.

## 🛠️ Решенные Проблемы (Knowledge Base)

### Исправление CSP и Медиа (TailwindBlog 2.0 Legacy)
Были решены проблемы с отображением `<iframe>` (YouTube) и локального видео через настройку `next.config.js` (в контексте перехода с Next на Analog).

### Ошибки Импорта
При разработке важно соблюдать иерархию путей:
- Из `pages/tags/[tag]` путь до компонентов: `../../../components/logo.component`.
- **Важно:** Использовать единственное число в именах файлов: `logo.component.ts`.

## 🛡️ Команды Стражей (CLI)
Для управления проектом используем:
- `gemini-cli`: Для аудита кода.
- `kilo.ai/cli`: Для решения сложных коллизий.
